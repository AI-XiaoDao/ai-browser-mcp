## 五分钟上手

1. 启动 **AI浏览器.exe**
2. `http://127.0.0.1:9222/health` → `status: ok`
3. Cursor 配置 `node mcp_bridge.js`（见配置说明书）
4. `browser_navigate` 打开目标 URL
5. 读操作用 sync-wait 默认（`browser_get_title`、`browser_evaluate`、`dom_*`）

```bash
node mcp_bridge.js --check
node tests/batch_test.js system
```

## 常用工具

- **导航**：`browser_navigate`、`browser_get_url`、`browser_reload`
- **读页面**：`browser_evaluate`、`browser_dom_query`、`browser_get_title`
- **填表**：`browser_fill_set_value`、`browser_fill_click`
- **网络**：`browser_network` list、`browser_collect` reverse_prepare
- **Hook**：`browser_inject` persist + `inject_id`
- **工作流**：`workflow_run` name=hello

POST body 不在网络层 → 用 `browser_network detail_enable` 开启详细记录

## Sync-Wait

默认一次返回；`async_only:true` 得 task_id；`max_ms` 调超时。单浏览器脚本**顺序**执行。

**同步等待白名单**：`browser_get_title`、`browser_evaluate`、`browser_dom_*`、`browser_fill_*`、`browser_cdp_call`、`browser_debugger_*`、`browser_get_text`、`browser_dom_get_html` 等，自动同步返回结果。

```json
// 同步等待 5 秒
{"name":"browser_evaluate","arguments":{"code":"1+1","max_ms":5000}}
// 异步模式：立即返回 task_id，稍后轮询
{"name":"browser_get_cookies","arguments":{"async_only":true}}
```

## 🛡️ 减少失败的实战技巧

### ① JSON 转义 — 最常见的失败原因

```bash
# ❌ 易失败：inline code 含引号/反斜杠 → shell 吞转义 → JSON 非法
curl -d '{"code":"document.querySelector(\"a[href*=\\\"/video/\\\"]\")"}' ...

# ✅ 方案A：code_base64（零转义风险，推荐）
CODE_B64=$(echo 'document.querySelector("a")' | base64 -w0)
curl -d '{"code_base64":"'$CODE_B64'","max_ms":5000}' ...

# ✅ 方案B：file 参数（从文件读JS）
echo 'document.querySelector("a")' > /tmp/script.js
curl -d '{"file":"/tmp/script.js","max_ms":5000}' ...

# ✅ 方案C：curl -d @file（JSON写入文件）
cat > /tmp/req.json << 'EOF'
{"jsonrpc":"2.0","id":"x","method":"tools/call","params":{"name":"browser_evaluate","arguments":{"code":"1+1","max_ms":5000}}}
EOF
curl -d @/tmp/req.json http://127.0.0.1:9222/mcp
```

**`code_base64` 和 `file` 参数适用于**：`browser_evaluate`、`browser_execute_js`、`browser_console_eval`。

### ② 自动启用 — 无需手动 enable

```json
// ❌ 旧方式：两步操作
{"name":"browser_collect","arguments":{"action":"console_enable"}}  // 先启用
{"name":"browser_collect","arguments":{"action":"console_get"}}    // 再查询

// ✅ 新方式：一步到位（首次自动启用）
{"name":"browser_collect","arguments":{"action":"console_get"}}

// browser_network list 同样自动启用
{"name":"browser_network","arguments":{"action":"list"}}
```

### ③ 同步等待 vs 异步轮询

| 场景 | 方式 | 示例 |
|------|------|------|
| 轻量读操作 | sync-wait（默认） | `browser_get_title`、`browser_evaluate "1+1"` |
| 重量操作 | async + 轮询 | `browser_get_source`、`browser_get_cookies` |
| 超时后继续 | `async_only:true` | 超时不报错，task_id 保留可后续查询 |
| CDP 命令 | `max_ms` 同步 | `browser_cdp_call {max_ms:20000}` |

```json
// 轮询异步结果
{"name":"mcp_result","arguments":{"request_id":"task_xxx","consume":true}}
```

### ④ 填表/DOM 操作 — 确认元素存在

```json
// ✅ 先检查再操作
{"name":"browser_dom_click","arguments":{"selector":"#submit-btn"}}  // 原生链：存在检查→点击
{"name":"browser_fill_exists","arguments":{"selector":"#login-form"}}
// 不存在时返回 "element not found" 而非静默失败
```

### ⑤ 网络抓包 — 一键开启

```json
// 开启详细模式（含 POST body）
{"name":"browser_network","arguments":{"action":"detail_enable"}}
// 操作页面...
// 查看结果（首次自动启用）
{"name":"browser_network","arguments":{"action":"list","limit":20}}
```

### ⑥ 爬虫提取 — 一步到位

```json
// 一站式爬虫：导航→等待→提取
{"name":"browser_scrape","arguments":{"url":"https://example.com","extract_selector":".content","max_ms":30000}}
// 结构化提取
{"name":"browser_extract","arguments":{"type":"links","limit":50}}
```

### ⑦ 进程管理 — Node.js 启停

```bash
node mcp_launcher.js start       # 启动
node mcp_launcher.js stop        # 安全关闭
node mcp_launcher.js status      # 查看状态
```

```json
// MCP 工具远程关闭
{"name":"browser_shutdown","arguments":{"confirm":true}}
```
