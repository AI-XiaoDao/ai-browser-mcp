# AI浏览器 — 客户使用手册（技能摘要）

> 完整版：`docs/客户使用手册.md` · 在线 `http://127.0.0.1:9222/docs/客户使用手册.md`

**面向客户**：帮用户用自然语言通过 Cursor 操控浏览器，无需手写 MCP 协议。

## 产品一句话

启动 AI浏览器.exe → 本机 MCP 服务（9222）→ Cursor/脚本调用 217 个 `browser_*` 工具自动化网页。

## 全功能开箱即用

**GitHub Release zip** 下载的 exe 已包含 **全部 217 个工具**（截图、CDP、调试器、指纹、拦截等），解压运行即可。

## 客户上手四步

1. 启动 **AI浏览器.exe**（关窗口≠退出，看托盘）
2. 打开 `http://127.0.0.1:9222/health` 确认 ok
3. 配置 Cursor `mcp.json` + `mcp_bridge.js`（欢迎页可一键复制）
4. 对 AI 说：「打开 https://example.com 并读标题」或「滚动列表采 JSON」

## 客户常用话术 → 工具

| 客户说 | 工具 |
|--------|------|
| 打开某网址 | `browser_navigate` |
| 当前标题/URL | `browser_get_title` / `browser_get_url` |
| 填输入框、点按钮 | `browser_fill_set_value` / `browser_fill_click` |
| 看网络请求 | `browser_network` list |
| 滚动采集 JSON | `dom_query` / `collect` / `evaluate` |
| 扫描 POST 加密字段 | `reverse_prepare` + `inject` Hook |
| 定位 sign 函数 | `debugger_*` 断点/栈/源码 |
| 登录填表导出 | `fill_*` + `workflow_run` |
| 截图 | `browser_screenshot` |

## 客户 FAQ

- **ECONNREFUSED**：先启动 exe
- **关窗口还能用**：托盘仍在跑 MCP
- **完全退出**：托盘右键退出
- **页面卡死 / 扫描 timeout**：`node scenarios/debugger_unfreeze.js`
- **POST 扫描**：`douyin_xhr_encrypt_scan.js --wait 20 --scroll`（勿并行 MCP 脚本）
- **高级工具报错**：确认使用的是 [GitHub Release](https://github.com/AI-XiaoDao/ai-browser-mcp/releases) 下载包并重启 exe
