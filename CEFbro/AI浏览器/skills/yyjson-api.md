# yyjson API 完整参考 — 火山视窗

## 概述

本项目提供两套 yyjson 封装，均在 `CEFbro\火山官方模块API\` 目录下：

| 文件 | 封装作者 | 定位 |
|------|---------|------|
| [w_yyjson.wsv](CEFbro\火山官方模块API\w_yyjson.wsv) | 飞翔的企鹅（官方） | 轻量封装，直接映射 yyjson C API |
| [JSON操作库_.wsv](CEFbro\火山官方模块API\JSON操作库_.wsv) | lyjdgkg（第三方） | 厚重封装，简化的属性读写 + JSON Pointer 支持 |

**选择原则**：
- **解析用（只读）** → 优先 `YYJSON只读对象类` / `YYJSON只读数组类`（官方，性能最高）
- **构造用（读写）** → 优先 `YYJSON对象类` / `YYJSON数组类`（官方，API 简洁）
- **频繁路径访问** → 用 `JSON对象类_` / `JSON数组类_`（第三方，支持 JSON Pointer `/path/to/node`）
- **MCP Server 当前使用** → `YYJSON只读对象类`（解析）+ `YYJSON对象类` / `YYJSON数组类`（构造）

### ❌ 错误1：只读对象用 mutable 方法
```火山
变量 解析结果 <类型 = YYJSON只读对象类>
解析结果.加入文本成员 ("key", "value")  // ❌ 只读类没有加入方法！
```
✅ 只读 = `YYJSON只读对象类`（仅解析），可变 = `YYJSON对象类`（可构造）

### ❌ 错误2：创建自文本带引号
```火山
对象.创建自文本 ("\"hello\"")   // ❌ 这会解析 "hello" 作为 JSON 文本值，非对象
```
✅ 构造对象必须 `创建自文本 ("{}")`，构造数组必须 `创建自文本 ("[]")`

### ❌ 错误3：忘记判空
```火山
变量 val <类型 = YYJSON只读对象类>
val = 解析结果.取对象 ("params")
val.取文本 ("key")  // ❌ 如果 "params" 不存在，val 为空，crash
```
✅ 用 MCP 的 `yyjson取文本(解析结果, "key")` 或手动判空

### ❌ 错误4：JSON Pointer 路径不以 / 开头
```火山
对象.取路径对象 ("user/name")   // ❌ 路径必须以 / 开头！
```
✅ 正确：`对象.取路径对象 ("/user/name")`

### ⚠️ 注意5：压缩模式是 MCP 默认
MCP Server 所有响应使用 `YYJSON格式化选项.压缩`（无换行缩进），减少 token 消耗。

### ⚠️ 注意6：查找参数的含义
第三方封装中的 `查找` 参数（默认 `真`）：
- `真`：先查找键是否存在，存在则更新，不存在则新建（安全，略慢）
- `假`：直接添加新节点，不检查重复（快，但有重复风险）

## 五、库选择决策树

```
需要做什么？
├── 只解析 JSON（不修改）──────────────→ YYJSON只读对象类 / YYJSON只读数组类
│   └── 需要按路径导航 ────────────────→ JSON对象类_解析 / JSON数组类_解析
│
├── 构造新 JSON ─────────────────────→ YYJSON对象类 / YYJSON数组类
│   └── 需要频繁路径操作 ──────────────→ JSON对象类_ / JSON数组类_
│
├── 遍历对象/数组成员 ─────────────────→ 枚举循环() + 取枚举值对象()
│
└── 简单取值/置值（不关心性能）────────→ JSON对象类_ / JSON数组类_
```
