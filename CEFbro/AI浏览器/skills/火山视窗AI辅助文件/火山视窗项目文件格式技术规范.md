## 1. 概述

### 1.1 文件类型说明

| 文件扩展名 | 文件类型 | 用途说明 |
|-----------|---------|---------|
| `.vsln` | 火山解决方案文件 (Volcanic Solution) | 组织和管理多个火山视窗项目的容器文件，定义项目间的层次结构和依赖关系 |
| `.vprj` | 火山项目文件 (Volcanic Project) | 定义单个 Windows 应用程序的完整配置，包括源文件、模块依赖、构建参数等 |

### 1.2 编码格式

**重要**: 所有 `.vprj` 和 `.vsln` 文件均采用 **UTF-8**编码格式

在读取或写入这些文件时，必须使用 **UTF-8** 编码，否则会导致解析错误。

### 1.3 文件格式特点

- **格式特点**: 采用键值对和块结构的混合语法
- **层次化结构**: 使用花括号 `{}` 定义嵌套块
- **标识符命名**: 使用小写字母和下划线的组合
- **字符串值**: 使用双引号包裹，路径使用反斜杠 `\`

## 2. .vsln 文件结构详解

### 2.1 基本结构

```
doc_format_version = <版本号>
name = <解决方案名称>
property.settings.active_project_name = <活动项目名称>

root_filter
{
    child_filter
    {
        filter<N>
        {
            // 项目引用或嵌套过滤器
        }
        
        num_child_filters = <子过滤器数量>
    }
    
    name = root
}
```

### 2.2 文件头信息

| 字段名 | 必需 | 说明 | 示例 |
|-------|------|------|------|
| `doc_format_version` | 是 | 文档格式版本号 | `1` |
| `name` | 是 | 解决方案显示名称 | `samples` |
| `property.settings.active_project_name` | 否 | 当前活动的项目名称 | `PDF创作` |

### 2.3 根过滤器 (root_filter)

`root_filter` 是解决方案的顶层容器，包含以下结构：

#### 2.3.1 子过滤器 (child_filter)

每个 `child_filter` 包含多个 `filter<N>` 节点（N 为数字序号）：

```
filter1
{
    build_order = <构建顺序>
    project_file_name = "<项目文件相对路径>"
    projects_dependent
    type = project
}
```

| 字段名 | 必需 | 说明 | 示例 |
|-------|------|------|------|
| `build_order` | 是 | 项目构建顺序号 | `0` |
| `project_file_name` | 是 | 项目文件路径（相对于解决方案） | `samples\\tab_demo\\tab_demo.vprj` |
| `projects_dependent` | 是 | 标记为项目依赖（无值） | - |
| `type` | 是 | 节点类型，固定为 `project` | `project` |

#### 2.3.2 嵌套过滤器

过滤器可以嵌套形成层次结构，支持分组管理：

```
filter19
{
    child_filter
    {
        filter1
        {
            build_order = 0
            project_file_name = "samples\\cef\\cef.vprj"
            projects_dependent
            type = project
        }
        
        filter2
        {
            child_filter
            {
                filter1
                {
                    build_order = 0
                    project_file_name = "samples\\mini_blink\\mini_blink.vprj"
                    projects_dependent
                    type = project
                }
                
                num_child_filters = <子项数量>
            }
            
            name = MiniBlink浏览器
        }
        
        num_child_filters = <子项数量>
    }
    
    collapsed = true
    name = 浏览器
}
```

| 字段名 | 必需 | 说明 | 示例 |
|-------|------|------|------|
| `name` | 是 | 分组显示名称 | `浏览器` |
| `collapsed` | 否 | IDE 中是否折叠显示 | `true` |
| `num_child_filters` | 是 | 直接子项数量 | `3` |

### 2.4 完整示例

```
doc_format_version = 1
name = samples
property.settings.active_project_name = PDF创作

root_filter
{
    child_filter
    {
        filter1
        {
            child_filter
            {
                filter1
                {
                    build_order = 0
                    project_file_name = "samples\\tab_demo\\tab_demo.vprj"
                    projects_dependent
                    type = project
                }
                
                filter2
                {
                    build_order = 0
                    project_file_name = "samples\\tab_demo2\\tab_demo.vprj"
                    projects_dependent
                    type = project
                }
                
                num_child_filters = 28
            }
            
            name = 窗口组件
        }
        
        filter10
        {
            child_filter
            {
                filter1
                {
                    build_order = 0
                    project_file_name = "samples\\aliyun_oss\\aliyun_oss.vprj"
                    projects_dependent
                    type = project
                }
                
                num_child_filters = 11
            }
            
            name = 第三方
        }
        
        num_child_filters = 13
    }
    
    name = root
}
```

## 3. .vprj 文件结构详解

### 3.1 基本结构

```
doc_format_version = <版本号>
project_identifier = "<项目标识符>"

root_filter
{
    child_filter
    {
        // 模块和类库分组
    }
    
    files
    {
        // 源文件列表
    }
    
    name = <项目名称>
    type = project
}

settings
{
    project
    {
        // 项目设置
    }
}
```

### 3.2 文件头信息

| 字段名 | 必需 | 说明 | 示例 |
|-------|------|------|------|
| `doc_format_version` | 是 | 文档格式版本号 | `1` |
| `project_identifier` | 是 | 项目类型标识符 | `wutao.vproject.pc.win.1` |

**项目标识符说明**:
- `wutao.vproject.pc.win.1` - Windows 桌面应用项目
- 该标识符用于火山 IDE 识别项目类型和加载相应的构建工具链

### 3.3 根过滤器 (root_filter)

#### 3.3.1 基本属性

| 字段名 | 必需 | 说明 | 示例 |
|-------|------|------|------|
| `name` | 是 | 项目显示名称 | `选择夹1` |
| `type` | 是 | 节点类型，固定为 `project` | `project` |
| `collapsed` | 否 | IDE 中是否折叠 | `true` |

#### 3.3.2 子过滤器结构 (child_filter)

用于组织系统类库和模块依赖：

```
filter1
{
    child_filter
    {
        filter1
        {
            collapsed = true
            files.num_files = 16
            group_file_name = "*plugins\\vprj_win\\classlib\\sys\\base\\base.vgrp"
            name = 视窗基本类
        }
        
        filter2
        {
            collapsed = true
            files.num_files = 4
            group_file_name = "*plugins\\vprj_win\\classlib\\sys\\mfc_ui_base\\mfc_ui_base.vgrp"
            name = MFC界面基本类
        }
        
        num_child_filters = <子项数量>
    }
    
    name = 模块
}
```

| 字段名 | 必需 | 说明 | 示例 |
|-------|------|------|------|
| `collapsed` | 是 | 是否折叠显示 | `true` |
| `files.num_files` | 是 | 类库包含的文件数 | `16` |
| `group_file_name` | 是 | 类库组文件路径 | `*plugins\\vprj_win\\classlib\\sys\\base\\base.vgrp` |
| `name` | 是 | 类库显示名称 | `视窗基本类` |

#### 3.3.3 类库路径规则

类库组文件路径以 `*plugins\\` 开头，表示火山 IDE 插件目录：

**系统类库路径模式**:
```
*plugins\\vprj_win\\classlib\\sys\\<分类>\\<类库名>\\<类库组文件>.vgrp
```

**常见类库路径**:

| 类库名称 | 路径 | 文件数 |
|---------|------|--------|
| 视窗基本类 | `*plugins\\vprj_win\\classlib\\sys\\base\\base.vgrp` | 16 |
| MFC界面基本类 | `*plugins\\vprj_win\\classlib\\sys\\mfc_ui_base\\mfc_ui_base.vgrp` | 4 |
| WTL界面基本类 | `*plugins\\vprj_win\\classlib\\sys\\wtl_ui_base\\wtl_ui_base.vgrp` | 3 |
| JSON数据访问支持 | `*plugins\\vprj_win\\classlib\\sys\\json\\w_json.vgrp` | 1 |
| Curl网络传输 | `*plugins\\vprj_win\\classlib\\sys\\curl\\w_curl.vgrp` | 1 |
| 火花代码编辑框 | `*plugins\\vprj_win\\classlib\\sys\\scintilla\\w_scintilla.vgrp` | 1 |
| ECharts图表组件 | `*plugins\\vprj_win\\classlib\\sys\\mfc_ui_echarts\\mfc_ui_echarts.vgrp` | 5 |
| WTL版Edge浏览框 | `*plugins\\vprj_win\\classlib\\sys\\ui_webview2\\wtl_ui_webview2.vgrp` | 1 |
| MFC扩展界面支持类库1 | `*plugins\\vprj_win\\classlib\\sys\\mfc_control_ext1\\mfc_control_ext1.vgrp` | 1 |
| WTL扩展界面支持类库1 | `*plugins\\vprj_win\\classlib\\sys\\wtl_control_ext1\\wtl_control_ext1.vgrp` | 1 |

### 3.4 源文件列表 (files)

定义项目包含的所有源代码文件：

```
files
{
    file1.file_name = "src\\main.wsv"
    file2.file_name = "main.wsv"
    num_files = <文件总数>
}
```

| 字段名 | 必需 | 说明 | 示例 |
|-------|------|------|------|
| `file<N>.file_name` | 是 | 源文件相对路径 | `src\\main.wsv` |
| `num_files` | 是 | 源文件总数 | `1` |

**文件命名规则**:
- 文件序号从 1 开始连续编号
- 路径使用反斜杠 `\\` 分隔
- 支持的文件类型：`.wsv` (火山源文件)

### 3.5 项目设置 (settings)

#### 3.5.1 基本项目设置

```
settings
{
    project
    {
        result_type = <输出类型>
        target_platform = <目标平台>
        out_file_name = "<输出文件名>"
        debug_command_line = "<调试命令行>"
        debug_working_dir = "<调试工作目录>"
        predefine_macros = "<预定义宏>"
        static_runtime_lib = <是否静态链接运行库>
        auto_support_high_dpi = <是否支持高DPI>
    }
}
```

| 字段名 | 必需 | 类型 | 说明 | 示例 |
|-------|------|------|------|------|
| `result_type` | 否 | integer | 输出类型：1=DLL, 2=EXE | `2` |
| `target_platform` | 否 | integer | 目标平台：1=Win32, 2=x64 | `1` |
| `out_file_name` | 否 | string | 输出文件路径 | `"..\\out\\$(p)\\test.dll"` |
| `debug_command_line` | 否 | string | 调试命令行参数 | `"..\\out\\$(p)\\test_call.exe"` |
| `debug_working_dir` | 否 | string | 调试工作目录 | `"..\\out\\$(p)"` |
| `predefine_macros` | 否 | string | 预定义宏 | `_WIN32` |
| `static_runtime_lib` | 否 | boolean | 是否静态链接运行库 | `false` |
| `auto_support_high_dpi` | 否 | boolean | 是否自动支持高DPI | `true` |

#### 3.5.2 设置项简写形式

简单设置可使用点号路径简写：

```
settings.project.result_type = 2
settings.project.predefine_macros = _WIN32
settings.project.auto_support_high_dpi = true
```

## 4. 解决方案与项目的关联配置

### 4.1 关联方式

解决方案通过 `project_file_name` 字段关联项目文件：

```
project_file_name = "samples\\tab_demo\\tab_demo.vprj"
```

**路径规则**:
- 相对于 `.vsln` 文件所在目录
- 使用双反斜杠 `\\` 作为路径分隔符
- 必须指向有效的 `.vprj` 文件

### 4.2 层次结构

解决方案可包含多层嵌套的项目分组：

```
root_filter
└── child_filter
    ├── filter1 (窗口组件)
    │   └── child_filter
    │       ├── filter1 (选择夹示例)
    │       │   ├── tab_demo.vprj
    │       │   ├── tab_demo2.vprj
    │       │   └── tab_demo3.vprj
    │       ├── filter2 (对话框示例)
    │       └── filter3 (列表视图示例)
    ├── filter2 (浏览器)
    │   └── child_filter
    │       ├── filter1 (CEF浏览器)
    │       ├── filter2 (MiniBlink浏览器)
    │       └── filter3 (Edge浏览器)
    └── filter3 (第三方)
```

### 4.3 构建顺序

`build_order` 字段定义项目的编译顺序：

```
build_order = 0  // 数字越小越先编译
```

**注意**: 当前所有示例项目的 `build_order` 均为 0，表示按默认顺序编译。

### 4.4 项目依赖

`projects_dependent` 标记表示项目间存在依赖关系：

```
filter1
{
    build_order = 0
    project_file_name = "base_lib\\base_lib.vprj"
    projects_dependent  // 标记此项被其他项目依赖
    type = project
}
```

### 4.5 活动项目配置

解决方案可指定当前活动的项目：

```
property.settings.active_project_name = PDF创作
```

该配置用于 IDE 在打开解决方案时自动激活指定项目。

## 5. 关键配置项说明

### 5.1 项目标识符 (project_identifier)

**作用**: 唯一标识项目类型，决定使用的构建工具链

**固定值**: `wutao.vproject.pc.win.1`

**AI 生成提示**: 创建 Windows 项目时必须使用此标识符

### 5.2 类库依赖配置

**核心类库** (几乎所有项目都需要):

```
// 视窗基本类库
group_file_name = "*plugins\\vprj_win\\classlib\\sys\\base\\base.vgrp"
files.num_files = 16

// 界面基本类库（二选一）
// MFC界面基本类
group_file_name = "*plugins\\vprj_win\\classlib\\sys\\mfc_ui_base\\mfc_ui_base.vgrp"
files.num_files = 4

// WTL界面基本类
group_file_name = "*plugins\\vprj_win\\classlib\\sys\\wtl_ui_base\\wtl_ui_base.vgrp"
files.num_files = 3
```

**常用功能类库**:

| 功能 | 类库路径 | 必需性 |
|------|---------|--------|
| JSON 处理 | `*plugins\\vprj_win\\classlib\\sys\\json\\w_json.vgrp` | 按需 |
| 网络传输 | `*plugins\\vprj_win\\classlib\\sys\\curl\\w_curl.vgrp` | 网络项目必需 |
| 代码编辑 | `*plugins\\vprj_win\\classlib\\sys\\scintilla\\w_scintilla.vgrp` | 编辑器项目必需 |
| 图表组件 | `*plugins\\vprj_win\\classlib\\sys\\mfc_ui_echarts\\mfc_ui_echarts.vgrp` | 图表项目必需 |
| Edge浏览框 | `*plugins\\vprj_win\\classlib\\sys\\ui_webview2\\wtl_ui_webview2.vgrp` | 浏览器项目必需 |
| 扩展界面支持 | `*plugins\\vprj_win\\classlib\\sys\\mfc_control_ext1\\mfc_control_ext1.vgrp` | 复杂界面按需 |
| 百度网盘支持 | `*plugins\\vprj_win\\classlib\\sys\\baidu_Netdisk\\w_baidu_Netdisk.vgrp` | 网盘项目必需 |

### 5.3 源文件配置

**最小项目配置**:
```
files
{
    file1.file_name = "src\\main.wsv"
    num_files = 1
}
```

**多文件项目**:
```
files
{
    file1.file_name = "src\\main.wsv"
    file2.file_name = "类_全局.wsv"
    file3.file_name = "类_窗口.wsv"
    num_files = 3
}
```

**命名约定**:
- 主程序文件：`src\\main.wsv` 或 `main.wsv`
- 类文件：`类_*.wsv`
- 窗口文件：`窗口_*.wsv`

### 5.4 项目设置项详解

#### 5.4.1 输出类型 (result_type)

- **类型**: integer
- **必需**: 否
- **取值**: 
  - `1` = 动态链接库 (DLL)
  - `2` = 可执行文件 (EXE)
- **默认**: 2 (EXE)

#### 5.4.2 目标平台 (target_platform)

- **类型**: integer
- **必需**: 否
- **取值**:
  - `1` = Win32 (32位)
  - `2` = x64 (64位)
- **默认**: 1 (Win32)

#### 5.4.3 输出文件名 (out_file_name)

- **类型**: string
- **必需**: 否
- **说明**: 自定义输出文件路径和名称
- **变量支持**:
  - `$(p)` = 平台 (win32/x64)
  - `$(r)` = 结果类型 (dll/exe)
- **示例**: `out_file_name = "..\\out\\$(p)\\test.dll"`

#### 5.4.4 调试配置 (debug_command_line, debug_working_dir)

- **debug_command_line**: 调试时启动的命令行
- **debug_working_dir**: 调试时的工作目录
- **典型用法**: 用于 DLL 项目的调试调用

#### 5.4.5 预定义宏 (predefine_macros)

- **类型**: string
- **必需**: 否
- **说明**: 编译时预定义的宏，多个宏用逗号分隔
- **示例**: `predefine_macros = "_WIN32, test1, test2"`

#### 5.4.6 运行库链接方式 (static_runtime_lib)

- **类型**: boolean
- **必需**: 否
- **说明**: 是否静态链接 C++ 运行库
- **默认**: false (动态链接)

#### 5.4.7 高DPI支持 (auto_support_high_dpi)

- **类型**: boolean
- **必需**: 否
- **说明**: 是否自动支持高DPI显示
- **默认**: false
- **推荐**: true (现代 Windows 应用)

### 5.5 分组配置 (collapsed)

**作用**: 控制 IDE 中节点的展开/折叠状态

```
collapsed = true   // 折叠
collapsed = false  // 展开（或不设置，默认展开）
```

**推荐配置**:
- 系统类库分组：`collapsed = true`
- 项目源文件分组：`collapsed = false`
