## ✨ 这是什么？

**AI浏览器 MCP Server** 是一款运行在 Windows 上的本地浏览器自动化服务。启动后在本机 `9222` 端口暴露 **Model Context Protocol (MCP)** 接口，为 AI 助手提供 **217 个** `browser_*` 工具——导航、填表、读 DOM、抓网络、工作流、CDP 断点等，开箱即用。

> 你只需在 Cursor 里说：「打开 example.com 并读取标题」—— AI 会自动调用 MCP 工具完成。

```mermaid
flowchart LR
  subgraph clients [AI 客户端]
    Cursor[Cursor / Claude]
    Script[Node 脚本]
  end
  Bridge[mcp_bridge.js]
  MCP[MCP Server\nHTTP + WebSocket]
  CEF[FBrowser CEF\n真实浏览器]

  Cursor --> Bridge --> MCP --> CEF
  Script --> MCP
```

## 🚀 快速开始

### 方式 A：使用成品（推荐新手）

1. 从 [Releases](https://github.com/AI-XiaoDao/ai-browser-mcp/releases) 下载 **`AI-Browser-MCP-x64-v2.6.0.zip`**（约 157MB，已排除编译中间产物）
2. 解压，双击 **`AI浏览器.exe`**
3. 浏览器打开 `http://127.0.0.1:9222/health`，确认 `"status":"ok"`
4. 配置 Cursor（仓库根目录 `.mcp.json` 可复制）：

```json
{
  "mcpServers": {
    "ai-browser": {
      "command": "node",
      "args": ["CEFbro/AI浏览器/mcp_bridge.js"],
      "env": {
        "AI_BROWSER_MCP_HTTP_POST": "http://127.0.0.1:9222/mcp"
      }
    }
  }
}
```

5. 自检：`node CEFbro/AI浏览器/mcp_bridge.js --check`

### 方式 B：从火山源码编译

**依赖**：Windows x64、[火山视窗 IDE](https://www.voldp.com/)（含 FBrowser CEF 模块）、Node.js（桥接脚本）

1. 打开 `CEFbro/AI浏览器/AI浏览器.vprj`，编译 **Release x64**
2. 运行时分发目录：`_int/AI浏览器/release/x64/linker/`（含 exe、dll、docs、workflows）
3. 打包 Release 时**排除** `linker/out/`（见下方「编译目录说明」）

## 📂 开源范围与火山编译目录

本仓库开源 **火山视窗 `.wsv` 源码**、**编译生成的 C++ 对照**及文档/脚本，**不包含** FBrowser CEF 闭源库与 `linker/out/` 中间产物。

### 仓库内有什么 / 没有什么

| 路径 | 入 Git | 说明 |
|------|--------|------|
| `CEFbro/AI浏览器/src/*.wsv` | ✅ | **开源核心** — 11 个 MCP 服务源文件 |
| `CEFbro/AI浏览器/generated-cpp/release-x64/` | ✅ | 火山自动生成的 **C++ 源码**（33 cpp + 218 h） |
| `CEFbro/AI浏览器/docs/`、`skills/` | ✅ | 客户手册、217 工具参考、火山 API 技能书 |
| `CEFbro/AI浏览器/mcp_*.js`、`workflows/` | ✅ | 桥接脚本、测试与工作流 JSON |
| `release/linker/` | ✅ | 配置包镜像（脚本/文档/工作流，无大二进制） |
| `.mcp.json` | ✅ | Cursor MCP 配置示例 |
| `_int/` 全部 | ❌ | 本地编译缓存（`.gitignore` 排除） |
| **`linker/out/`** | ❌ | **不是 C++ 源码**，仅为 `.obj`/`.pch`，禁止提交或打包 |
| `*.exe` / `libcef.dll` 等 | ❌ | 走 [Releases](https://github.com/AI-XiaoDao/ai-browser-mcp/releases) |
| FBrowser CEF 模块 | — | 随火山 IDE 安装，**不在本仓** |

### 火山 IDE 编译后 `_int` 目录（Release x64）

编译 `AI浏览器.vprj` 后，产物位于 `CEFbro/AI浏览器/_int/AI浏览器/release/x64/`：

```
_int/AI浏览器/release/x64/
├── project/          ← ★ 自动生成的 C++ 源码（.cpp / .h / makefile）
├── linker/           ← ★ 运行时分发目录（给用户用的成品）
│   ├── AI浏览器.exe
│   ├── *.dll / CEF 运行时
│   ├── docs/ workflows/ mcp_*.js
│   └── out/          ← ✗ 编译中间产物（.obj / .pch），不是源码，勿打包
├── compiler/         ← 编译器临时目录（通常为空）
└── debuger/          ← 调试器辅助目录（通常为空）
```

**哪个才是 C++ 源码？** → **`project/`** 目录（仓库内同步为 [`generated-cpp/release-x64/`](CEFbro/AI浏览器/generated-cpp/release-x64/)）。

火山将 `src/*.wsv` 翻译为 MSVC 可编译的 C++，例如 `vpkg_main.cpp`、`vpkg_MCP_Server.cpp` 及大量 `vcls_*.h`。文件头标注 `Generated by Volcano Development Platform`。**二次开发请以 `src/*.wsv` 为准**；C++ 为编译对照，每次 Release 后同步更新。

**`linker/out/` 是什么？** → 不是 C++ 源码，是链接前的**目标文件**（`.obj`、`.pch`、`make_params.txt`），体积约 400MB，**成品 zip 必须排除**。

```mermaid
flowchart LR
  WSV["src/*.wsv\n开源主体"]
  PROJ["project/*.cpp\n自动生成 C++"]
  OUT["linker/out/*.obj\n中间产物"]
  LINK["linker/*.exe\n运行成品"]

  WSV -->|火山翻译| PROJ
  PROJ -->|MSVC 编译| OUT
  OUT -->|链接 + 复制 CEF| LINK
```

## ❓ 常见问题

| 问题 | 处理 |
|------|------|
| `/health` 失败 | 确认 exe 已启动；检查 9222 端口占用 |
| Cursor 连不上 MCP | `node mcp_bridge.js --check`；确认 `AI_BROWSER_MCP_HTTP_POST` |
| 工具调用超时 | 增大 `mcp_config.json` 中 `default_timeout_ms` |
| POST body 抓不到 | 默认网络层不记录 POST 正文，见 `skills/场景与Hook测试.md` |
| 成品 zip 很大 | 正常；勿把 `linker/out/` 打进包（见 `release/pack-release.ps1`） |

更多 FAQ 见 [OPEN_SOURCE.md#常见问题](OPEN_SOURCE.md#常见问题faq)。
