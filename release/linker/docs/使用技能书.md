## 1. 五分钟上手

| 步骤 | 操作 |
|------|------|
| 1 | 启动 **AI浏览器.exe**（关闭窗口会最小化到托盘，MCP 仍运行） |
| 2 | 浏览器访问 `http://127.0.0.1:9222/health` 确认 `status: ok` |
| 3 | Cursor：配置 `mcp_bridge.js`（见《MCP工具配置说明书》） |
| 4 | 调用 `browser_navigate` 打开网页 |
| 5 | 可选：`node run_all_tests.js --quick` 冒烟 |

```bash
node mcp_bridge.js --check          # 连接自检
node scenarios/douyin_xhr_encrypt_scan.js --help   # POST 加密扫描
```

## 2. 连接方式

| 方式 | 地址 | 适用 |
|------|------|------|
| HTTP JSON-RPC | `POST http://127.0.0.1:9222/mcp` | 脚本、`mcp_client.js`、curl |
| WebSocket | `ws://127.0.0.1:9222` | 原生 MCP 客户端 |
| Cursor stdio | `node mcp_bridge.js` | IDE 集成 |
| 文档 | `http://127.0.0.1:9222/docs/` | 人类阅读 |

启动后自动设置环境变量：`AI_BROWSER_MCP_URL` / `PORT` / `HEALTH`。exe 同目录生成 `mcp_connect.json`。

## 3. 常用 MCP 调用

### 导航

```json
{"method":"tools/call","params":{"name":"browser_navigate","arguments":{"url":"https://www.douyin.com/"}}}
```

### 读页面（默认 sync-wait，一次返回）

```json
{"name":"browser_get_title","arguments":{}}
{"name":"browser_evaluate","arguments":{"code":"document.title"}}
{"name":"browser_dom_query","arguments":{"selector":"h1"}}
```

### 填表

```json
{"name":"browser_fill_set_value","arguments":{"selector":"#user","value":"test"}}
{"name":"browser_fill_click","arguments":{"selector":"button[type=submit]"}}
```

### 网络日志

```json
{"name":"browser_network","arguments":{"action":"list","limit":200}}
{"name":"browser_collect","arguments":{"action":"reverse_prepare","clear":true}}
```

> **注意**：`browser_network` **不记录 POST body**。抓 POST 参数须 `browser_inject` persist Hook，见 `scenarios/douyin_xhr_encrypt_scan.js`。

## 4. Sync-Wait（推荐）

v2.5+ 读/写工具默认 **sync-wait**：一次 `tools/call` 内完成，响应可含 `_sync_waited: true`。

| 参数 | 效果 |
|------|------|
| （默认） | 白名单工具自动等待结果 |
| `async_only: true` | 立即返回 `task_id`，再 `mcp_result` |
| `sync_wait: true` | 非白名单也同步等待 |
| `max_ms: N` | 自定义超时毫秒 |

**白名单**：`get_title`、`evaluate`、`console_eval`、`dom_*`、`fill_exists`、`fill_attr_get`、`get_frames`；debugger 全套（VIP）。

## 5. 场景脚本（顺序执行，勿并行）

| 脚本 | 用途 |
|------|------|
| `run_all_tests.js` | 全量回归 |
| `scenarios/run_all_scenarios.js` | Hook + 自动化 + 断点 |
| `scenarios/douyin_xhr_encrypt_scan.js` | XHR POST 加密字段扫描（见下表） |
| `scenarios/reverse_analyze.js [url]` | 页面 JS + 网络脚本清单 |
| `scenarios/debugger_unfreeze.js` | 解除 debugger 卡死 |

```bash
node scenarios/douyin_xhr_encrypt_scan.js --help
node scenarios/douyin_xhr_encrypt_scan.js --wait 20 --min 3 --scroll
node scenarios/douyin_xhr_encrypt_scan.js --skip-navigate --json report.json
```

| 选项 | 说明 |
|------|------|
| `--wait N` | 最长等待秒数（默认 15） |
| `--min N` | 捕获 N 条 POST 后提前结束 |
| `--scroll` | 等待时串行滚动（勿并发 MCP） |
| `--skip-navigate` | 跳过导航，在当前页扫描 |
| `--json FILE` | 保存 JSON 报告 |

轮询超时：先 `debugger_unfreeze.js`；脚本内 evaluate 轮询为 20s 短超时，连续 2 次失败即退出。

环境变量：`AI_BROWSER_MCP_HOST`、`AI_BROWSER_MCP_PORT`（默认 9222）。

## 6. 工作流 workflow

定义在 `workflows/*.json`，编译后复制到 `linker/workflows/`。

```json
{"name":"workflow_run","arguments":{"name":"hello"}}
{"name":"workflow_list","arguments":{}}
```

步骤支持 `{ "tool": "...", "args": {} }`、`{ "delay_ms": 500 }`、`{ "wait_async": true, "max_ms": 45000 }`。

## 7. 逆向 / Hook 速查

| 目标 | 做法 |
|------|------|
| 开网络+控制台 | `browser_collect` → `reverse_prepare` |
| 持久 JS Hook | `browser_inject` `{ type:"js", persist:true, inject_id:"唯一ID" }` |
| 导航拦截 | `browser_intercept` → `navigate_block` / `navigate_redirect` |
| VIP 改响应 | `browser_intercept` → `modify`（需 `vip_code`） |
| 断点调试 | `browser_debugger_flow` 或 `debug_prepare` 指南 |

Hook 脚本须用 `if (!window.__XXX__) { ... }` 包裹，避免重复注入。

---

## 8. 限制与注意

- ⛔ 勿用 `browser_create` / `browser_close` 等远程创建关闭（GUI 管理）
- 单浏览器 + MCP 全局锁：**脚本勿并行**
- MCP 全局锁：**`--scroll` 与轮询须串行**，勿 `setInterval` 并发 MCP 调用
- VIP 工具需 `mcp_config.json` 配置 `vip_code`
- 指纹/代理等修改后可能 `needs_reload: true`
