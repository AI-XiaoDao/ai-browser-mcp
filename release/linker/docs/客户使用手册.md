## 一、产品是什么？

**AI浏览器** 是一款内置浏览器的桌面程序。启动后会在本机开放 **MCP 服务**（默认端口 `9222`），让 **Cursor、Claude Desktop** 或您自己的程序像「远程操控浏览器」一样：

- 打开指定网页
- 自动填表、点击按钮
- 读取页面文字与元素
- 查看网络请求记录
- （VIP）截图、改指纹、断点调试、拦截修改接口等

您不需要手写复杂代码——在 Cursor 里用自然语言描述任务，AI 会调用 **217 个**已封装好的浏览器工具完成操作。

## 二、您能得到什么？

| 能力 | 说明 | 是否需要 VIP |
|------|------|--------------|
| 打开/刷新/后退网页 | 日常浏览自动化 | 否 |
| 读标题、读元素、执行简单 JS | 信息采集 | 否 |
| 自动填表、点击 | 登录、下单、录入 | 否 |
| 网络请求列表 | 看 URL、状态码（不含 POST 正文） | 否 |
| 工作流一键执行 | 多步骤任务编排 | 否 |
| 截图 / PDF | 页面留存 | 是 |
| CDP 深度调试 / 断点 | 逆向分析 | 是 |
| 指纹 / 代理 / 资源拦截修改 | 防检测、改包 | 是 |

> **说明**：关闭程序主窗口后，程序会缩到**系统托盘**，MCP 服务通常仍在运行。要彻底退出请在托盘图标上右键选择退出。

## 三、系统要求

- Windows 10 / 11（64 位）
- 已安装 **Node.js**（仅在使用 Cursor 桥接 `mcp_bridge.js` 时需要）
- 本机可访问互联网（访问目标网站时）
- 防火墙允许本机 `127.0.0.1:9222`（默认仅本机访问，安全）

---

## 四、安装与首次启动

1. 解压或安装发布包，找到 **`AI浏览器.exe`**
2. 双击启动，等待主窗口出现
3. 用浏览器打开：`http://127.0.0.1:9222/health`  
   - 看到 `"status":"ok"` 表示服务正常
4. 打开欢迎页控制台：`http://127.0.0.1:9222/`  
   - 可查看工具数量、复制 Cursor 配置、搜索工具名

**exe 同目录重要文件：**

| 文件 | 作用 |
|------|------|
| `mcp_config.json` | 端口、日志、VIP 授权等配置 |
| `mcp_connect.json` | 启动后自动生成，含连接地址 |
| `mcp_bridge.js` | 给 Cursor 用的桥接脚本 |
| `docs/` | 本手册及配置说明（随安装包附带） |

## 五、三种使用方式

### 方式 A：在 Cursor 里用自然语言（推荐）

适合：不懂编程的客户、日常「帮我打开某网站并操作」。

1. 保持 **AI浏览器.exe** 运行
2. 按 [配置说明书](./MCP工具配置说明书.md) 配置 Cursor 的 `mcp.json`
3. 重启 Cursor，在对话中直接说，例如：

| 您可以说 | AI 会做什么 |
|----------|-------------|
| 「打开百度」 | `browser_navigate` 打开 https://www.baidu.com |
| 「当前页面标题是什么」 | `browser_get_title` |
| 「在搜索框输入天气并点击搜索」 | `fill_set_value` + `fill_click` |
| 「列出最近的网络请求」 | `browser_network` list |
| 「刷新页面」 | `browser_reload` |

4. 自检连接（可选）：

```bash
node mcp_bridge.js --check
```

### 方式 B：欢迎页控制台

适合：快速查看服务状态、浏览全部工具、复制配置。

- 地址：`http://127.0.0.1:9222/`
- 文档：`http://127.0.0.1:9222/docs/`

### 方式 C：自有脚本 / 其他 MCP 客户端

适合：集成到公司内部系统。

- HTTP 接口：`POST http://127.0.0.1:9222/mcp`
- 协议：标准 MCP JSON-RPC（`tools/call`）
- 示例见 [使用技能书](./使用技能书.md) 第 3 节

## 六、Cursor 接入（分步说明）

### 第 1 步：确认 Node.js

命令行执行 `node -v`，能显示版本号即可。

### 第 2 步：编辑 Cursor 配置

在用户目录或项目下创建/编辑 **`.cursor/mcp.json`**（路径以 Cursor 版本为准）：

```json
{
  "mcpServers": {
    "ai-browser": {
      "command": "node",
      "args": ["您的安装路径/mcp_bridge.js"],
      "env": {
        "AI_BROWSER_MCP_HTTP_POST": "http://127.0.0.1:9222/mcp"
      }
    }
  }
}
```

将 `args` 中的路径改为您实际的 `mcp_bridge.js` 位置（一般在 exe 同目录）。

也可在欢迎页 `http://127.0.0.1:9222/` 点击 **「复制 Cursor 配置」** 一键复制。

### 第 3 步：重启 Cursor

配置保存后完全退出并重新打开 Cursor，在 MCP 工具列表中应能看到 `ai-browser` 相关工具。

### 第 4 步：试一句

在对话输入：**「用浏览器打开 https://www.example.com 并告诉我标题」**。

## 七、常见业务场景

### 7.1 日常打开网页

对 AI 说：「打开抖音」或「导航到 https://www.douyin.com」。

### 7.2 自动填表登录

对 AI 说：「在 `#username` 填入 xxx，在 `#password` 填入 yyy，点击登录按钮」。  
（需页面有稳定的选择器；复杂站点可让 AI 先 `browser_dom_query` 分析结构。）

### 7.3 查看接口 / 网络活动

对 AI 说：「开启网络日志并列出最近 50 条请求」。  
注意：默认**看不到 POST 请求体**。技术人员可用附带脚本（在 exe 同目录或源码 `scenarios/`）：

```bash
node scenarios/douyin_xhr_encrypt_scan.js --wait 20 --min 3 --scroll
node scenarios/douyin_xhr_encrypt_scan.js --skip-navigate   # 当前已在抖音页时
```

若脚本报 **timeout**，先执行 `node scenarios/debugger_unfreeze.js` 解除页面卡顿，再重试。

### 7.4 批量重复操作（工作流）

对 AI 说：「执行工作流 hello」→ 对应 `workflow_run`。  
预置工作流在 `workflows/` 目录，可按需定制 JSON 步骤链。

### 7.5 页面卡住 / 调试后无响应

在 exe 目录执行：

```bash
node scenarios/debugger_unfreeze.js
```

或在对话中让 AI 调用 `browser_restore_gui`、`browser_reload`。

## 八、VIP 授权说明

部分高级功能需要 **FBrowser VIP 授权码**：

1. 用记事本打开 exe 同目录 **`mcp_config.json`**
2. 填写：`"vip_code": "您的授权码"`
3. **保存后重启** AI浏览器.exe
4. 再试截图、CDP、指纹等工具

未配置时，调用 VIP 工具会提示：`VIP控制器不可用,需要VIP授权`。

授权与购买咨询见文末 **技术支持**。

## 九、修改配置时要注意

| 操作 | 说明 |
|------|------|
| 改端口 | 修改 `mcp_config.json` 的 `port`，重启后 Cursor 的 URL 也要一起改 |
| 开网络日志 | 默认已开启 `enable_network_log: true` |
| 改配置后 | **必须重启** AI浏览器.exe 才生效 |
| 详细字段 | 见 [MCP工具配置说明书](./MCP工具配置说明书.md) |

## 十、常见问题（客户版）

| 问题 | 解决办法 |
|------|----------|
| Cursor 里工具用不了 | 先启动 AI浏览器.exe → 运行 `node mcp_bridge.js --check` |
| 提示连接被拒绝 | 程序未启动或端口被占用；检查 9222 端口 |
| 关了窗口还能用吗？ | 可以，窗口关闭后缩到托盘，MCP 一般仍在 |
| 怎么完全退出？ | 托盘图标右键 → 退出 |
| AI 说找不到浏览器 | 等几秒再试，或打开一次主窗口让浏览器实例创建 |
| VIP 功能全部报错 | 检查 `mcp_config.json` 里 `vip_code` 是否填写并重启 |
| 网络列表是空的 | 需要先打开网页产生请求；确认 `enable_network_log` 为 true |
| 文档网页打不开 | 确认安装目录下有 `docs` 文件夹（重新安装/编译发布包） |
| 多个脚本同时跑乱了 | 同一台机器**不要并行**跑多个自动化脚本，请顺序执行 |
| 扫描脚本报 timeout | 先 `node scenarios/debugger_unfreeze.js`，再重试；勿同时开多个 MCP 调用 |
| 抖音/接口加密参数怎么看？ | 见 7.3 节 `douyin_xhr_encrypt_scan.js`，或联系技术支持 |
